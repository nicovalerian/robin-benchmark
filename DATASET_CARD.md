# ROBIN Benchmark Dataset

**ROBIN** (Robustness Of instruction-following under Bahasa-Indonesia/eNglish
code-mixing) is a 4-level Indonesian–English code-mixing benchmark for measuring
how robustly an LLM follows verifiable instructions as input style shifts from
formal Bahasa Indonesia toward heavily code-mixed "Jaksel"-style language.

Each **base instruction** is rewritten into four perturbation levels (L0–L3) that
hold the *task and its verifiable constraints fixed* while increasing
code-mixing. A robust model should produce equally constraint-compliant,
semantically faithful answers at every level.

## Dataset structure

One JSON object per **base instruction** (`data/processed/robin_dataset.jsonl`).
With the full release, 750 base instructions × 4 levels = **3,000 evaluation
prompts**.

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Stable unique id, `robin_00000`..`robin_00749` (contiguous, sorted) |
| `category` | string | One of the five task categories (below) |
| `original_instruction` | string | The source Indonesian Alpaca instruction (unique per row) |
| `original_input` | string | Optional context passage; appended to the prompt in Phase 2 |
| `gold_response` | string | Reference answer generated at `temperature=0.0` against the constrained instruction |
| `constraints` | list | Three verifiable constraints (keyword, length, format) — see below |
| `perturbations` | object | The four level texts: `level_0_clean`, `level_1_mild`, `level_2_jaksel`, `level_3_adversarial` |
| `perturbation_meta` | object | Engine metadata for the rewrite |

### `constraints[]` objects

| Field | Description |
|-------|-------------|
| `constraint_type` | `keyword` \| `length` \| `format` |
| `requirement` | Human-readable Indonesian requirement, **embedded verbatim at the end of every level text** |
| `verification_regex` | Regex used by the Phase 3 constraint checker |
| `target_value` | keyword: list of words · length: `[min, max]` words · format: format id |

### Perturbation levels

| Level | Name | What changes |
|-------|------|--------------|
| **L0** | Formal Indonesian | Control baseline — standard Bahasa Indonesia |
| **L1** | Lexical borrowing | Absorbed English nouns only, no morphology |
| **L2** | Morphological fusion | Indonesian affixes on English verb stems (`diextract`, `mengupdate`) + English discourse connectors |
| **L3** | Intra-sentential switching | L2 + clause-level switching + ID-root/EN-suffix + Indonesian phonological respelling + colloquial particles |

The three constraint requirement lines are preserved **verbatim** across all four
levels, so the only thing varying within a base instruction is the code-mixing
style.

## How it was built

- **Source**: [`ilhamfadheel/alpaca-cleaned-indonesian`](https://huggingface.co/datasets/ilhamfadheel/alpaca-cleaned-indonesian), balanced-subset sampled with `random_seed=42`.
- **Constraints** are derived deterministically (keyword/length from the gold response, format by category preference) and injected into the instruction text.
- **Perturbations + gold** are generated via DigitalOcean Serverless Inference using `gemma-4-31B-it` (gold at `temperature=0.0`, L1–L3 at `temperature=0.4`), then L2/L3 receive deterministic seeded post-processing (IndoCollex substitutions, discourse particles).

See the repository `README.md` and `CLAUDE.md` for full pipeline details.

## Intended use & evaluation

ROBIN is paired with a three-stage evaluation pipeline (constraint pass rate +
ROUGE-L + IndoBERT BERTScore). The **primary robustness metric is Worst-Level
Robustness (WLR)** = min composite score over L0–L3, reported beside the mean
level score.

### Known limitations (read before drawing conclusions)

- **Self-preference**: `gold_response` is generated by `gemma-4-31B-it`, which is
  also an evaluated model. It shows ~0.25 semantic self-inflation and is
  therefore **excluded from headline rankings**. Prefer a held-out reference if
  you re-run generation.
- **CPR is dominated by length**: keyword and format compliance saturate near
  1.0; only the length constraint discriminates. Report the three constraint
  types separately.
- **Creative-writing semantic scores** are depressed by single-reference
  divergence, not necessarily by worse instruction-following.
- The perturbation ladder is **not guaranteed monotone** (L0 ≥ L1 ≥ L2 ≥ L3);
  some models bottom out at L2.

## Reproducibility

`random_seed=42` fixes subset selection and post-processing RNG, but LLM rewrites
run at `temperature=0.4` — re-running Phase 1 selects the same source items but
produces different L0–L3 text. **The committed `robin_dataset.jsonl` is the
canonical reproducible artifact.** Validate any copy with
`python scripts/validate_dataset.py`.
